## 研究の概要

Wikipediaのカテゴリ階層には、クラス・サブクラスやクラス・インスタンスなど、様々な関係が存在しています。本研究では、大規模言語モデル（LLM）を活用し、これらを整理・分類することで、包含関係の推移律が成立する有用な階層構造のみを抽出する「Wikipediaカテゴリオントロジー」の構築・更新を効率化する手法を提案しています。

## 公開プロンプトの内容

本リポジトリでは、論文内で提案している2段階の分類プロセスに対応するプロンプトを公開しています。

- **[pipe1.txt](pipe1.txt): カテゴリの種類と構成要素への分割プロンプト**
  - カテゴリ名から構造情報（中心となる名詞、制約など）を抽出し、カテゴリを `SetCategory` や `TopicCategory` などの型に分類します。
- **[pipe2.txt](pipe2.txt): カテゴリの階層関係の分類プロンプト**
  - 親子関係にあるカテゴリペアの型と制約情報を入力とし、その関係における推移律の成立可否を判定し、カテゴリペア間の関係（`subclassOf`, `instanceOf`, `geography` など）を9種類に分類します。
